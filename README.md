# HA 集成云海湾门禁系统

这个仓库用于把已经验证可用的本地云海湾门禁“虚拟室内机”原型，逐步迁移到 Home Assistant 中。最终目标是在 HA Yellow 上用软件方式替代现有室内机，实现门禁监控、室外机呼入、接听、挂断、呼叫中解锁，以及后续的物业中心通话等功能。

当前本地原型已经围绕 1 栋 A 座完成了主要功能验证：可以查看各号门口机实时画面，监听室外机呼叫，在呼叫弹窗中显示视频，并在被呼叫状态下执行解锁。这个仓库会以这些已验证能力为基础，分阶段迁移为 HA 可长期运行、可配置、可集成 Dashboard 的方案。

## 目标

- 将云海湾门禁系统接入 Home Assistant。
- 在 HA 中提供一个虚拟室内机后端服务，常驻监听呼叫和管理视频流。
- 在 Dashboard 中用一个轻量卡片承载入口，点击后显示完整功能界面。
- 室外机呼入时自动弹出小窗口，支持查看视频、接听、挂断和解锁。
- 支持按楼栋选择门禁配置，后续补全不同楼栋的门口机 IP。
- 将已验证的本地 Python 后端和 Web UI 逐步拆分成 HA Add-on、Custom Integration 和 Lovelace 卡片。

## 当前迁移方向

优先采用三层结构：

1. HA Add-on：运行门禁协议、视频流、音频和解锁相关的常驻后端。
2. HA Custom Integration：向 Home Assistant 暴露设备、实体、服务和事件。
3. Lovelace Custom Card：负责 Dashboard 中的虚拟室内机交互界面。

这种方式可以让复杂的网络监听、端口占用、RTP/RTSP/音频处理留在后端服务里，同时让 HA 只负责状态、自动化、服务调用和前端展示。

当前已经开始第一阶段开发：

- 根目录提供 `repository.yaml`，让 Home Assistant 可以按 Add-on 仓库识别。
- `addons/yunhai_intercom/` 已创建最小 Add-on 骨架。
- Add-on 目前先加载本机 IP、本机 ID、楼栋、中心地址和物业中心地址配置。
- 内置 1 栋 A 座 8 个已知门口机，其他楼栋先保留空 IP 占位。
- 后端入口会常驻运行，下一步会把旧原型里的 UDP 监听和会话逻辑迁移进来。
- 协议层已经开始迁移，当前包含呼叫中解锁包、监控发现包、监控请求/保活包、视频分片解析和 JPEG 重组。

## 功能范围

计划迁移的功能包括：

- 门口机监控：主动查看已配置门口机的实时画面。
- 呼叫监听：持续监听室外机呼叫，并中断当前监控流切换到呼叫画面。
- 呼叫弹窗：呼入时显示视频、接听、挂断、解锁按钮。
- 呼叫中解锁：仅在被呼叫或通话状态下执行已验证可用的解锁流程。
- 物业中心呼叫：后续根据抓包数据加入主动呼叫物业中心能力。
- 楼栋配置：按楼栋加载门口机 IP、本机 IP、房号和本机 ID。
- HA 事件：把呼入、接听、挂断、解锁、视频状态等事件暴露给自动化。

暂不纳入本仓库的功能：

- 原室内机自带的无用硬件功能。
- 安防和燃气报警器逻辑。
- 原机系统设置界面的完整复刻。

安防和燃气报警器后续更适合通过 ESP32 接入 HA，由 Home Assistant 自身的安防、传感器和自动化能力处理。

## 迁移计划

第一阶段：整理现有本地原型，把门禁协议、配置、视频流和 UI 职责拆清楚。

第二阶段：创建 HA Add-on 骨架，将当前 Python 后端改造成可在 HA 中常驻运行的服务。

第三阶段：创建 HA Custom Integration，提供设备、实体、服务调用和事件订阅。

第四阶段：创建 Lovelace Custom Card，复刻当前已经确认的虚拟室内机界面和弹窗交互。

第五阶段：补齐音频通话、物业中心呼叫、更多楼栋配置、铃声选择和 HA 自动化联动。
