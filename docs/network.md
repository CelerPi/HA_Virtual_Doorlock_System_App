# 网络连接方式

## 门禁网络参数

当前已知参数：

```text
子网掩码：255.255.248.0
网关地址：192.168.16.1
中心地址：192.168.16.2
物业中心机：192.168.23.255
```

门禁通信使用：

```text
UDP 10000
UDP 10008
```

Add-on 使用 `host_network: true`，因此它会直接绑定 HA 主机网络上的 `local_ip:10000` 和 `local_ip:10008`。

Add-on HTTP API 默认监听：

```text
TCP 8099
```

如果 `api_host` 设置为 `0.0.0.0`，同一网络中的设备可以访问该端口。解锁/接听接口需要 `api_token`。

## 推荐连接方式

将 HA Yellow 或运行 HA OS/Supervised 的主机接入门禁所在网络，并给它配置一个门禁网段 IP。

如果目标是完全替代原室内机，推荐让 HA 主机使用原室内机身份：

```text
IP：192.168.16.64
ID：00010116010
房号：1601
```

注意：不要让原室内机和 HA 主机同时使用同一个 IP。需要并行测试时，应让 HA 使用临时 IP，并确认门口机会把呼叫发到这个 IP；否则呼入监听可能收不到包。

## 与旧 Unlock 原型的关系

旧原型可用的关键条件仍然适用：

- 本机 IP 必须真实配置在当前网卡上。
- UDP `10000` 和 `10008` 不能被其他进程占用。
- 解锁依赖室外机正在呼叫本室内机的会话状态。
- `b7000600` 是呼叫会话中确认有效的解锁命令。
- 主动监控和呼叫视频共用端口，后续接监控功能时呼叫必须抢占监控。

## 简单连通性检查

在 HA 主机上检查：

```bash
ip addr
```

确认能看到 `local_ip`。

如果有 SSH 终端，可以检查 UDP 端口占用：

```bash
ss -lunp | grep -E ':10000|:10008'
```

Add-on 启动后，这两个端口应由 Add-on 进程占用。
